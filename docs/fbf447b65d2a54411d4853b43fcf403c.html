supervisor 的安装和使用

简介

Supervisor是用Python开发的一套通用的进程管理程序，能将一个普通的命令行进程变为后台daemon，并监控进程状态，异常退出时能自动重启。

安装

Supervisor 适用于 Python 3 版本 3.4 或更高版本，以及 Python 2 版本 2.7

  1. 通过 pip 安装

pip install supervisor

  1. 导出配置

# root 用户 echo_supervisord_conf > /etc/supervisord.conf # sudo 用户 sudo sh -c "echo_supervisord_conf > /etc/supervisord.conf"

  1. 修改配置

sudo sed -i 's/tmp\/supervisor.sock/var\/tmp\/supervisor.sock/g' /etc/supervisord.conf sudo sed -i 's/tmp\/supervisord.pid/var\/tmp\/supervisord.pid/g' /etc/supervisord.conf

  1. 配置守护进程

sudo vim /usr/lib/systemd/system/supervisord.service # 文件内容如下： [Unit] Description=Supervisor daemon [Service] Type=forking ExecStart=/usr/local/bin/supervisord ExecStop=/usr/local/bin/supervisorctl $OPTIONS shutdown ExecReload=/usr/bin/supervisorctl $OPTIONS reload KillMode=process Restart=on-failure RestartSec=42s [Install] WantedBy=multi-user.target

  1. 启动服务

# 重载服务列表 sudo systemctl daemon-reload # 启动 sudo systemctl start supervisord # 开机自启 sudo systemctl enable supervisord

使用

常用配置说明

# x 为项目名称，名称中不得包含冒号或方括号 [program:x] # process_name 进程名称，%(program_name)s代表项目名称，%(process_num)02d代表生成两位数字 process_name=%(program_name)s_%(process_num)02d # command 执行命令 command=pwd # numprocs 进程数量 numprocs=4 # numprocs_start 进程的起始编号 numprocs_start=0 # priority 优先级，程序在启动和关闭顺序中的相对优先级，较低的优先级（首先启动，最后关闭），较高的优先级（最后启动，首先关闭） priority=999 # autostart 自动启动，启动supervisor的时候自动启动 autostart=true # autorestart 自动重启 autorestart=true # user 指定用户执行操作 user=www # redirect_stderr 输出supervisord其标准输出文件描述符 redirect_stderr=true # stdout_logfile 指定日志文件 stdout_logfile=/www/supervisor.log # 项目组管理，x为组名称 [group:x] # 项目名称，中间用逗号隔开 programs=bar,baz # 优先级，非必填，默认999 priority=999

详细配置说明

[program:cat] command=/bin/cat # 运行的命令 process_name=%(program_name)s # 进程名称 numprocs=1 # 进程数量 directory=/tmp # 命令执行目录 umask=022 # 表示进程的umask八进制数，例如002、022 priority=999 # 优先级 autostart=true # 自动启动，在启动supervisord时自动启动 autorestart=unexpected # 自动重启 startsecs=10 # 启动后程序需要保持运行以认为启动成功的总秒数 startretries=3 # 尝试启动次数 exitcodes=0 # 退出错误码 stopsignal=TERM # 停止信号，可以是TERM, HUP, INT, QUIT, KILL, USR1, or USR2 stopwaitsecs=10 # 停止等待的秒数，超时会通过SIGKILL杀死进程 stopasgroup=false # 停止整个进程组 killasgroup=false # 杀死整个进程组 user=chrism # 使用指定用户启动进程 redirect_stderr=false # 不建议设置 stdout_logfile=/a/path # 标准输出日志文件 stdout_logfile_maxbytes=1MB # 标准输出日志文件大小，单位可以为：KB、MB、GB，如果为0则不限制日志大小 stdout_logfile_backups=10 # 保留日志文件数量，如果为0则不保留 stdout_capture_maxbytes=1MB stdout_events_enabled=false stderr_logfile=/a/path stderr_logfile_maxbytes=1MB stderr_logfile_backups=10 stderr_capture_maxbytes=1MB stderr_events_enabled=false environment=A="1",B="2" serverurl=AUTO

示例

go 应用

[program:http-go] process_name=%(program_name)s_%(process_num)02d command=/home/xiaocai/code/test/go-test/main autostart=true autorestart=true user=root numprocs=1 redirect_stderr=true stdout_logfile=/home/xiaocai/code/test/go-test/main.log

laravel 应用

[program:laravel-worker] process_name=%(program_name)s_%(process_num)02d command=php /home/forge/app.com/artisan queue:work sqs --sleep=3 --tries=3 autostart=true autorestart=true user=www numprocs=2 redirect_stderr=true stdout_logfile=/home/forge/app.com/worker.log
